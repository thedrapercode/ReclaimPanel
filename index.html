ReclaimPanel

Open-source ESP32 firmware for driving salvaged P20 triangle RGB LED panels with xLights and FPP. Free to use, free to modify.

v1.0 - Stable ESP32 WROOM-32 P20 Triangle Panels xLights + FPP MIT License

Install Firmware

Connect your ESP32 via USB, then click the button below. No software to download - installs directly from your browser.

Requires Chrome or Edge browser (Web Serial API). Firefox and Safari are not supported. Manual install instructions below.

What You Get

Full Color BCM Rendering

Binary Code Modulation cycles 4 bit-planes at 133 Hz, giving 4096 colors and smooth gradients instead of just on/off.

Web Interface

Built-in web UI for test colors, brightness, orientation, BCM speed, panel arrangement, diagnostics, and OTA updates.

xLights + FPP Ready

Receives DDP packets from Falcon Player. Set up your sequences in xLights and stream them live to your panels.

Up to 4 Panels

Drive 1 to 4 panels from a single ESP32. Arrange them in any grid layout using the web UI - no code changes needed.

OTA Updates

Update firmware over WiFi from the web UI. No need to reconnect USB after the initial flash.

Diagnostics

Live signal strength, DDP frame rate, heap usage, MAC address, and an optional debug log - all in the browser.

What You Need

Item Notes
ESP32 (various Models Work) 30-pin board, D0WD-V3 chip recommended. Most generic "ESP32 Dev Module" boards work but was verified on WROOM-32.
P20 Triangle RGB Panels Grouptek brand, 20x20 pixels (400 triangles). These are the salvaged real estate sign panels this firmware was built for.
5V Power Supply Dedicated supply for the panels. Budget ~2-3A per panel at full white. Do NOT power panels from the ESP32 USB port.
20-pin Ribbon Cable HUB16 style. Comes with the panels. One per panel in a chain.
Raspberry Pi 4 running FPP Falcon Player receives sequences from xLights and streams DDP to the ESP32. FPP 9.x or later.
xLights on a PC or Mac Free sequencing software. Used to build your light shows and upload them to FPP.
Chrome or Edge browser Required for the browser installer (Web Serial API). Any browser works for the ESP32 web interface after install.
Important: The 5V panel supply ground must be connected to the ESP32 GND pin. Without a shared ground the panels will not respond.

How to Install

  1. Install the USB driver (if needed)

    Most ESP32 dev boards use a CH340 or CP2102 USB chip. If your computer does not recognize the ESP32 when plugged in, download and install the driver for your chip before continuing. Windows 10/11 usually installs CH340 automatically.

  2. Connect your ESP32 via USB

    Use a data-capable USB cable (not a charge-only cable). Open Chrome or Edge on your computer.

  3. Click Install Firmware at the top of this page

    Your browser will ask you to select the serial port your ESP32 is connected to. Choose the port labeled with your ESP32's USB chip (CH340, CP2102, or similar). The installer will erase the ESP32 and flash ReclaimPanel automatically. This takes about 30 seconds.

  4. Connect to the setup WiFi network

    After flashing, the ESP32 starts a setup access point. On your phone or computer, connect to WiFi network: ReclaimPanel-Setup with password reclaim1

  5. Open the web interface

    Open a browser and go to reclaimpanel.local or 192.168.4.1 . You will see the ReclaimPanel setup page.

  6. Configure your settings

    Under the Network tab: enter your WiFi name, password, and a static IP (default 10.0.0.101 works for most setups). Under the Panels tab: set the number of panels and the start channel to match your xLights model. Click Save and Restart.

  7. Wire your panels

    Connect the 20-pin ribbon cable from the ESP32 breakout to Panel 0 IN. For multiple panels, daisy-chain OUT to IN. Connect the 5V supply to the panels and connect the supply GND to ESP32 GND. See the full wiring guide for pin details.

  8. Set up xLights and FPP

    In xLights, create a DDP controller pointed at your ESP32 IP with the correct start channel and channel count (1200 per panel). Create a Matrix model: 20x20 per panel, Top Right, Horizontal, Don't Zig Zag. Upload sequences to FPP using FPP Connect, then play from FPP. See the full xLights and FPP guide.

Test without FPP: Use the Test tab in the web interface to send solid colors and run the Chase animation. If Chase works but FPP does not, the issue is in your FPP or xLights configuration, not the firmware.

Wiring Quick Reference

Pin 1 DR1 -> GPIO 25 Pin 2 DG1 -> GPIO 26 Pin 3 DB1 -> GPIO 27 Pin 4 DR2 -> GPIO 14 Pin 5 DG2 -> GPIO 33 Pin 6 DB2 -> GPIO 13 Pin 7 DR3 -> GPIO 23 Pin 8 DG3 -> GPIO 19 Pin 9 DB3 -> GPIO 5 Pin 10 DR4 -> GPIO 17 Pin 11 DG4 -> GPIO 16 Pin 12 DB4 -> GPIO 4 Pin 13 DR5 -> GPIO 2 Pin 14 DG5 -> GPIO 15 Pin 15 DB5 -> GPIO 18 Pin 16 GND -> GND Pin 17 CLK -> GPIO 22 Pin 18 GND -> GND Pin 19 LAT -> GPIO 21 Pin 20 OE -> GPIO 3
DG2 uses GPIO 33, not GPIO 12. GPIO 12 is a strapping pin that causes boot failures when driven HIGH at startup.

Manual Install (Arduino IDE)

If the browser installer does not work, you can flash using Arduino IDE 2.x.

  1. Install Arduino IDE 2.x

    Download from arduino.cc/en/software

  2. Add ESP32 board support

    File > Preferences > Additional Board Manager URLs, add:
    https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json
    Then Tools > Board > Board Manager, search for esp32 and install.

  3. Download the firmware source

    Download the repository from GitHub and open ReclaimPanel/ReclaimPanel.ino in Arduino IDE.

  4. Set board settings

    Tools > Board: ESP32 Dev Module
    Partition Scheme: Minimal SPIFFS (1.9MB APP with OTA / 128KB SPIFFS)
    CPU Frequency: 240 MHz
    Flash Size: 4MB (32Mb)
    Upload Speed: 921600

  5. Upload

    Select your ESP32 COM port under Tools > Port, then click Upload.

Frequently Asked Questions

Will this work with standard HUB75 panels?

No. This firmware is specifically for P20 triangle panels with HUB16 connectors (5 parallel shift register chains). It will not work with HUB75 panels and vice versa.

My panels light up but show wrong colors on the bottom half.

The bit mapping for shift register groups 2-4 (rows 9-20) may need hardware verification for your specific panel batch. Try toggling Flip Rows and Flip Columns in the Display tab. This is a known area for further testing with different panel revisions.

FPP can ping the ESP32 but no DDP packets arrive.

Check that the start channel in the FPP output configuration exactly matches what is set in the ESP32 web UI Panels tab. A mismatch causes all packets to be silently discarded. Also confirm the FPP output type is DDP, not E1.31 or artnet.

I see flickering in BCM color mode.

Open the Display tab in the web interface and drag the BCM Cycle Speed slider to the left (lower microseconds = faster refresh). Try 200-300 us for flicker-free output. If you see corrupted pixels at low values, increase it slightly.

Can I run more than 4 panels from one ESP32?

The current firmware supports up to 4 panels per ESP32. For larger installations, use multiple ESP32 units with separate DDP addresses and combine them as separate controllers in xLights.

The browser installer does not see my ESP32.

Make sure you are using Chrome or Edge (not Firefox or Safari). Confirm the USB cable supports data transfer. Install the CH340 or CP2102 driver if your OS does not recognize the device automatically.

About This Project

The Draper Family Light Show is a free community event run by the entire Draper Family in Kenton, Ohio. Our charity light show was gifted 60 older P20 triangle LED panels from a local real estate company that was upgrading to newer displays. What started as a salvage project turned into a deep dive when we discovered there was no compatible open source controllers for these panels anywhere.

Every existing solution... WLED, SmartMatrix, and the HUB75 ESP32 library.... targets newer panels that use dedicated row address lines and multiplexing in HUB75. These older panels use 5 independent 80-bit shift register chains sharing a single CLK, LAT, and OE line with no row addressing at all... UGH! No datasheet exists for them so after countless hours of hardware testing and a ton of firmware versions, the protocol was fully reverse engineered. It came down to the clock... once the 80-bit chain with a single shared latch was cracked, everything clicked into place. We decided to open source the result so nobody else has to go through the same process.
